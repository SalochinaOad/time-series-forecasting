# Full Stack Time Series Forecasting Masterclass  
A complete end‑to‑end repository for mastering classical, machine learning, deep learning, and advanced forecasting techniques in Python.

---

## 📘 Course Overview  
This repository contains all code, datasets, and notebooks used in the **Full Stack Time Series Forecasting Masterclass**.  
The course is structured into **4 learning phases**, guiding you from foundations to advanced forecasting and anomaly detection.

You will learn:

- Classical statistical models (ARIMA, SARIMA, SARIMAX)  
- Machine learning forecasting workflows  
- Deep learning architectures (LSTM, GRU, CNN)  
- Prophet modeling  
- Financial forecasting with RNNs  
- Time series anomaly detection  
- End‑to‑end project structure

## ⚙️ Installation & Setup  

Clone the repository:

```bash
git clone https://github.com/SalochinaOad/full-stack-time-series-forecasting
cd full-stack-time-series-forecasting
```

Install dependencies:

```bash
pip install -r setup/requirements.txt
```

Recommended environment:

- Python **3.10+**
- IDE: VS Code, JupyterLab, or PyCharm  
- Key libraries: pandas, numpy, statsmodels, scikit‑learn, matplotlib, seaborn

## 🚀 Recommended Workflow (4‑Phase Learning System)

To maximize retention, follow this flow for every module:

### **1. [Lecture Phase](ca://s?q=Explain_the_Lecture_Phase)**  
Watch the video lesson to see the concepts applied to real‑world scenarios.  
For deeper theory, you may optionally read the corresponding chapter in:  
**Practical Time Series Analysis & Forecasting: A User-Friendly Guide to Mastering End-to-End Modeling in Python**  
This book is *recommended*, not required — the course and notebooks contain everything needed.

### **2. [Hands‑On Phase](ca://s?q=Explain_the_Hands_On_Phase)**  
Open the corresponding Jupyter Notebook and run the code step‑by‑step.

### **3. [Challenge Phase](ca://s?q=Explain_the_Challenge_Phase)**  
Each notebook includes a **“Try it yourself”** section.  
Attempt these exercises before checking the solutions.

### **4. [Review Phase](ca://s?q=Explain_the_Review_Phase)**  
Revisit key plots, errors, and model outputs to reinforce understanding.
